Frequently asked questions

Everything you need to know about using CV Studio.

If you’re wondering how exporting works, whether it’s ATS‑safe, or how much you can customise, you’re in the right place.

Is my CV really ATS‑friendly?

Yes – as long as you export correctly.

▼
CV Studio uses semantic HTML, a clear single column structure, and standard section labels like “Experience”, “Education”, and “Skills”. When you export via your browser’s print dialog to PDF, all text stays selectable, so ATS software can read it just like a Word document.

How do I export to PDF?

Use your browser’s print function.

▼

Open the builder, click the “Export PDF” button above the preview, or press Ctrl + P / Cmd + P.

  • Select “Save as PDF” as the destination.
  • Turn off headers/footers in the print options.
  • Enable background graphics if you want accent colours.

Can I use this on mobile?

Yes, it’s responsive.

▼
The layout is built with Tailwind CSS and is fully responsive. You can fill in the guided steps on your phone or tablet. For the cleanest PDF export, we still recommend a laptop or desktop.

Can I change fonts or colours?

A little bit in the UI, fully in code.

▼

In the builder, you can switch accent colours between blue, slate, and emerald. If you’re comfortable editing code, you can open the HTML files and adjust the Tailwind config or font imports for a fully custom look.

Is my data stored anywhere?

In this version, everything is local.

▼
This static version of CV Studio runs fully in your browser. There is no backend or database connected, so your information is not uploaded or stored on a server. If you refresh the page, you’ll need to re‑enter your details.

Need a more advanced version?

This project is built with HTML, Tailwind CSS, and vanilla JavaScript. You can easily extend it with:

  • LocalStorage to save multiple CVs in the browser
  • A simple backend (Node/Express, Firebase, Supabase) to store profiles
  • Multiple templates and languages per user
